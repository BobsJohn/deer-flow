## 为什么

管理后台已有一期基础框架（侧边栏、用户管理、审核页），但体验上有明显缺口：概览太简陋看不到趋势、角色权限未落地（developer/user 无区别）、操作无日志、驳回后作者无法重新提交。

## 变更内容

1. **操作日志** — 新增 audit_log 表 + 后端记录谁何时做了什么 + 前端查看
2. **角色权限落地** — developer 可管理自己的 Skill（编辑/发布），user 只读
3. **我的技能** — developer 角色可见"我的技能"入口，管理自己发布的 Skill
4. **重新提交审核** — 作者在编辑页可将 draft 状态的 Skill 重新提交为 review
5. **概览页增强** — 增加新增趋势（7 日新增技能/用户）、待审核数量突显

## 功能

### 新增功能
- `audit-log`: 操作日志后端 + 前端查看
- `my-skills`: developer 专属"我的技能"页面
- `resubmit`: 驳回后重新提交审核

### 修改功能
- `admin-ui`: 概览页加趋势数据
- `skill-create`: user 不能创建/编辑 Skill（仅查看）
