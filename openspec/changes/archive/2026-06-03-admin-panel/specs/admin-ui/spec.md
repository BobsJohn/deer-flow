# 管理后台 UI 规格说明

## 新增需求

### 需求: 侧边栏布局

#### 场景: 进入管理后台

- **当** 管理员点击菜单中的「后台管理」
- **那么** 跳转到 `/marketplace/admin`
- **并且** 页面左侧显示侧边栏，右侧显示内容区
- **并且** 侧边栏包含：概览、用户管理、资产审核

### 需求: 概览页

#### 场景: 展示平台数据

- **当** 管理员访问 `/marketplace/admin`
- **那么** 展示统计卡片网格（用户数、技能数、待审核、执行次数）

### 需求: 用户管理页

#### 场景: 用户列表

- **当** 管理员访问 `/marketplace/admin/users`
- **那么** 展示表格：姓名 / 邮箱 / 角色 / 所属团队 / 操作
- **并且** 行内可切换角色（下拉选择 user/developer/admin）
- **并且** 点击确认后立即生效

### 需求: 资产审核页

#### 场景: 审核列表

- **当** 管理员访问 `/marketplace/admin/reviews`
- **那么** 展示待审核 Skill 列表（名称、描述、版本、提交时间）
- **并且** 每行有「通过」「驳回」按钮

#### 场景: 通过

- **当** 管理员点击「通过」
- **那么** 调用审核 API，更新状态为 published
- **并且** 技能从待审核列表移除

#### 场景: 驳回

- **当** 管理员点击「驳回」
- **那么** 弹出输入框填写驳回原因
- **并且** 调用审核 API，更新状态为 draft
