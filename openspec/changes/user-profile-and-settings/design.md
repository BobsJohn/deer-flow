# 设计

## 上下文

目前右上角只有导航链接（市场、团队），没有任何用户信息。后端 User 表已有 `avatar` 字段但从未使用。

## 目标/非目标

**目标：**
- 右上角显示用户头像 + 名称
- 下拉菜单含个人信息编辑（头像切换）、密码修改、退出登录
- 8 个内置头像用 emoji（去除了首字母生成方案，用 emoji 更简单且不需要 CDN）

**非目标：**
- 不上传自定义头像（V1 简配）
- 不做全局 user profile 页面（只做弹窗）

## 决策

### 决策1：内置头像用 emoji 字符串存储

User 表 avatar 字段存 emoji 字符（如 `"🦊"`），前端按 emoji 渲染。不需要 CDN、不需要文件上传，零依赖。

8 个头像：🦊 🐼 🐨 🦁 🐰 🐱 🐶 🦄

### 决策2：用户菜单作为独立组件

`<UserMenu />` 组件挂在 `<MarketplaceHeader>` 的 `rightSlot` 槽位里，不影响 header 的复用。

### 决策3：个人信息和修改密码分开两个弹窗

一个弹窗做太多事会变得复杂。个人信息弹窗专注于展示/编辑资料，密码修改弹窗独立。
