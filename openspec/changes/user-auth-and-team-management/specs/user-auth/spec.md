# 用户认证规格说明

## 新增需求

### 需求: 用户注册

#### 场景: 新用户注册成功

- **当** 用户提交注册表单（name + email + password + team_id）
- **那么** 系统创建用户记录，密码 SHA-256 哈希存储
- **并且** 返回 JWT token + 用户信息

#### 场景: 邮箱已存在

- **当** 用户使用已注册的 email 提交注册
- **那么** 返回 409 冲突错误，提示"邮箱已被注册"

### 需求: 用户登录

#### 场景: 登录成功

- **当** 用户使用 email + password 登录  
- **那么** 校验密码哈希，返回 JWT token（有效期 24h）+ 用户信息

#### 场景: 登录失败

- **当** 用户输入错误密码
- **那么** 返回 401，提示"邮箱或密码错误"

### 需求: 当前用户信息

#### 场景: 请求用户信息

- **当** 前端发送携带 Authorization: Bearer <token> 的请求
- **那么** 后端解析 token，返回当前用户信息（id, name, email, role, team_id）

#### 场景: token 过期

- **当** token 已过期
- **那么** 后端返回 401，前端重定向到登录页

### 需求: 鉴权中间件

#### 场景: 未认证请求

- **当** 请求未携带有效 token 且不是公开路由（/login, /register）
- **那么** 返回 401 Unauthorized
